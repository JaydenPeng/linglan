# 日志系统使用说明

## 概述

日志系统分为两种类型:
1. **API日志** - 记录所有API调用的info和error信息
2. **组件日志** - 记录组件保存操作的日志

## 使用方法

### 1. 导入日志工具

```typescript
import { logger } from '@/utils/logger'
```

### 2. 记录API日志

在API调用时记录日志:

```typescript
// 记录成功的API调用
logger.logApi({
  method: 'POST',
  url: '/api/generate-image',
  level: 'info',
  message: '图片生成请求成功'
})

// 记录失败的API调用
logger.logApi({
  method: 'POST',
  url: '/api/generate-image',
  level: 'error',
  error: '网络请求失败: 超时'
})
```

### 3. 记录组件日志

在组件保存操作时记录日志:

```typescript
// 保存图片参数
logger.logComponent({
  component: 'ImageParamsPanel',
  action: '保存图片参数',
  data: {
    aspect_ratio: '16:9',
    force_single: true
  }
})

// 保存视频参数
logger.logComponent({
  component: 'VideoParamsForm',
  action: '保存视频配置',
  data: {
    duration: 5,
    resolution: '1080p'
  }
})
```

## 日志页面

在导航栏点击"日志"图标可以查看日志页面,页面包含两个Tab:

- **API日志** - 显示所有API调用的info和error日志
- **组件日志** - 显示所有组件保存操作的日志

每个Tab都有独立的清空按钮,可以单独清空对应类型的日志。

## 示例集成

### 在API拦截器中集成

```typescript
// 在axios拦截器中
axios.interceptors.response.use(
  (response) => {
    logger.logApi({
      method: response.config.method?.toUpperCase() || 'GET',
      url: response.config.url || '',
      level: 'info',
      message: `请求成功 (${response.status})`
    })
    return response
  },
  (error) => {
    logger.logApi({
      method: error.config?.method?.toUpperCase() || 'GET',
      url: error.config?.url || '',
      level: 'error',
      error: error.message
    })
    return Promise.reject(error)
  }
)
```

### 在组件中集成

```typescript
// 在保存按钮的点击事件中
const handleSave = () => {
  // 保存逻辑...

  // 记录日志
  logger.logComponent({
    component: 'SettingsPanel',
    action: '保存设置',
    data: {
      theme: 'dark',
      language: 'zh-CN'
    }
  })
}
```
