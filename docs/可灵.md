# Omni-Video 创建任务接口文档

## 网络协议
- **协议**：https
- **请求地址**：`/v1/videos/omni-video`
- **请求方法**：POST
- **请求格式**：`application/json`
- **响应格式**：`application/json`

---

## 请求头（Headers）
| 字段名 | 值 | 描述 |
| :--- | :--- | :--- |
| `Content-Type` | `application/json` | 数据交换格式 |
| `Authorization` | 鉴权信息 | 鉴权信息，参考接口鉴权 |

---

## 请求体（Request Body）

| 字段名 | 类型 | 必填 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- | :--- |
| `model_name` | string | 否 | `kling-video-o1` | 模型名称。<br>枚举值：`kling-video-o1`, `kling-v3-omni` |
| `multi_shot` | boolean | 否 | `false` | 是否生成多镜头视频。<br>• `true` 时，`prompt` 参数无效，且不支持设定首尾帧生成视频。<br>• `false` 时，`shot_type` 及 `multi_prompt` 参数无效。 |
| `shot_type` | string | 否 | 空 | 分镜方式。<br>• 枚举值：`customize`, `intelligence`。<br>• 当 `multi_shot` 为 `true` 时，**必填**。 |
| `prompt` | string | 否 | 空 | 文本提示词，可包含正向/负向描述。支持通过 `<<<>>>` 格式指定主体/图片/视频（如 `<<<element_1>>>`）。<br>• 长度 ≤ 2500字符。<br>• 当 `multi_shot=false` **或** (`multi_shot=true` 且 `shot_type=intelligence`) 时，**不得为空**。 |
| `multi_prompt` | array | 否 | 空 | 各分镜信息（提示词、时长）。格式为对象数组：<br>`[{"index": int, "prompt": "string", "duration": "5"}]`<br>• 最多支持6个分镜，最少1个。<br>• 每个分镜内容 ≤ 512字符。<br>• 各分镜时长 ≥ 1，总和 = 任务总时长。<br>• 当 `multi_shot=true` 且 `shot_type=customize` 时，**不得为空**。 |
| `image_list` | array | 否 | 空 | 参考图列表，支持Base64或URL（需公网可访问）。可用作主体/场景/风格参考，或作为首/尾帧。<br>• 格式：`[{"image_url": "url", "type": "first_frame"}]` (`type` 可选 `first_frame`/`end_frame`，不填则为普通参考图)<br>• 仅尾帧时 **必须** 同时有首帧。<br>• 格式支持：jpg/jpeg/png。<br>• 文件 ≤ 10MB，尺寸 ≥ 300px，宽高比在 1:2.5 ~ 2.5:1 之间。<br>• 数量限制：有参考视频时 ≤ 4；无参考视频时 ≤ 7。<br>• O1模型：图片超2张时不支持设置首尾帧。 |
| `element_list` | array | 否 | 空 | 参考主体列表，基于主体库ID。格式：`[{"element_id": long}]`<br>• 数量限制（与参考图片数量累加）：<br>  - 有参考视频时，总数 ≤ 4（且不支持视频角色主体）。<br>  - 无参考视频时，总数 ≤ 7。<br>• 模型限制：<br>  - 首帧视频时，最多3个主体。<br>  - 首尾帧视频时，`kling-v3-omni` 最多3个主体，`kling-video-o1` **不支持**主体。 |
| `video_list` | array | 否 | 空 | 参考视频（通过URL）。可作为特征参考或待编辑视频。格式：<br>`[{"video_url": "url", "refer_type": "base", "keep_original_sound": "yes"}]`<br>• `refer_type`：`feature`(特征参考) / `base`(待编辑)。<br>• `keep_original_sound`：`yes`/`no`。<br>• 当有参考视频时，`sound` 参数值只能为 `off`。<br>• 限制：至多1段，≤200MB，MP4/MOV格式。<br>• 时长 ≥ 3s（上限视模型而定），分辨率 720px~2160px。 |
| `sound` | string | 否 | `off` | 生成视频时是否生成声音。<br>枚举值：`on`, `off` |
| `mode` | string | 否 | `pro` | 生成视频的模式。<br>• `std`：标准模式（性价比高）<br>• `pro`：专家模式（高品质） |
| `aspect_ratio` | string | 否 | 空 | 视频画面纵横比。<br>• 枚举值：`16:9`, `9:16`, `1:1`<br>• **必填**：当未使用首帧参考或视频编辑功能时。 |
| `duration` | string | 否 | `5` | 生成视频时长（秒）。<br>• 枚举值：3~15的整数（步长为1）<br>• 使用视频编辑功能（`refer_type=base`）时，此参数无效，输出时长与输入视频相同。 |
| `watermark_info` | object | 否 | 空 | 是否生成含水印结果。<br>格式：`{"enabled": boolean}` (`true`生成/`false`不生成) |
| `callback_url` | string | 否 | 空 | 任务结果回调通知地址。服务端会在任务状态变更时主动通知。 |
| `external_task_id` | string | 否 | 空 | 自定义任务ID，单用户下需保证唯一性，支持通过该ID查询任务。 |

# Omni-Video 查询任务（单个）接口文档

## 网络协议
- **协议**：https
- **请求地址**：`/v1/videos/omni-video/{id}`
- **请求方法**：GET
- **请求格式**：`application/json`
- **响应格式**：`application/json`

---

## 请求头（Headers）
| 字段名 | 值 | 描述 |
| :--- | :--- | :--- |
| `Content-Type` | `application/json` | 数据交换格式 |
| `Authorization` | 鉴权信息 | 鉴权信息，参考接口鉴权 |

---

## 请求路径参数（Path Parameters）

| 字段名 | 类型 | 必填 | 默认值 | 描述 |
| :--- | :--- | :--- | :--- | :--- |
| `task_id` | string | 否 | 无 | 文生视频的任务ID。<br>• 作为请求路径参数直接填写在URL中。<br>• 与 `external_task_id` 两种查询方式**二选一**。 |
| `external_task_id` | string | 否 | 无 | 文生视频的自定义任务ID。<br>• 即创建任务时填写的 `external_task_id`。<br>• 与 `task_id` 两种查询方式**二选一**。 |

> **注意**：`{id}` 在请求路径中是一个占位符，实际请求时需要替换为具体的 `task_id` 或 `external_task_id` 值。

---

# 接口鉴权

- Step-1：获取 AccessKey + SecretKey
- Step-2：您每次请求API的时候，需要按照固定加密方法生成API Token
	○ 加密方法：遵循JWT（Json Web Token, RFC 7519）标准
	○ JWT由三个部分组成：Header、Payload、Signature

```python
import time
import jwt

ak = "" # 填写access key
sk = "" # 填写secret key

def encode_jwt_token(ak, sk):
    headers = {
        "alg": "HS256",
        "typ": "JWT"
    }
    payload = {
        "iss": ak,
        "exp": int(time.time()) + 1800, # 有效时间，此处示例代表当前时间+1800s(30min)
        "nbf": int(time.time()) - 5 # 开始生效的时间，此处示例代表当前时间-5秒
    }
    token = jwt.encode(payload, sk, headers=headers)
    return token

api_token = encode_jwt_token(ak, sk)
print(api_token) # 打印生成的API_TOKEN
```
