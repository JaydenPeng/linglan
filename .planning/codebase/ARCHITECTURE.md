# 架构

**分析日期：** 2026-03-19

## 模式概览

**整体模式：** Electron 桌面应用

**关键特征：**
- 主进程 + 渲染进程分离架构
- 基于 Chromium 的跨平台桌面应用
- 简单的单窗口应用结构

## 层级

**主进程层：**
- 目的：应用生命周期管理、窗口创建、系统集成
- 位置：`main.js`
- 包含：Electron 应用启动逻辑、BrowserWindow 创建
- 依赖：`electron` 模块
- 被使用：应用启动时由 Electron 框架调用

**渲染进程层：**
- 目的：UI 展示和用户交互
- 位置：`index.html`
- 包含：HTML 标记、页面结构
- 依赖：主进程加载此文件
- 被使用：用户界面呈现

## 数据流

**应用启动流程：**

1. Electron 框架启动应用
2. `main.js` 中的 `app.whenReady()` 回调被触发
3. `createWindow()` 函数执行，创建 BrowserWindow 实例
4. 窗口加载 `index.html` 文件
5. 渲染进程启动，显示 UI

**用户交互流程：**

1. 用户在 UI 中进行操作
2. 渲染进程处理事件
3. 可通过 IPC 与主进程通信（当前未实现）

**状态管理：**
- 无集中状态管理
- 应用状态由 Electron 框架管理
- 窗口配置（宽度、高度）在 `main.js` 中硬编码

## 关键抽象

**BrowserWindow 抽象：**
- 目的：代表应用窗口
- 示例：`d:/Code/front/linglan/main.js` 中的 `win` 对象
- 模式：工厂函数 `createWindow()` 创建窗口实例

**应用生命周期：**
- 目的：管理应用启动和关闭
- 示例：`app.whenReady()` 钩子
- 模式：事件驱动

## 入口点

**应用入口：**
- 位置：`main.js`
- 触发：npm start 或 electron 命令
- 职责：初始化 Electron 应用、创建主窗口、管理应用生命周期

**UI 入口：**
- 位置：`index.html`
- 触发：主进程加载此文件
- 职责：定义应用 UI 结构、显示欢迎信息

## 错误处理

**策略：** 最小化处理

**模式：**
- 无显式错误处理
- 依赖 Electron 框架的默认错误处理
- 无 try-catch 块或错误回调
