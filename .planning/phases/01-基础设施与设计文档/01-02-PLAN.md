**安全红线：** AK/SK 只存在于主进程内存和 electron-store 加密存储中。渲染进程只接收布尔状态，密钥不经过 IPC 传回渲染进程。

## 2. IPC 通道定义

| 通道名 | 方向 | 入参 | 返回值 | 说明 |
|--------|------|------|--------|------|
| `config:save` | renderer→main | `SaveConfigPayload` | `IpcResult<void>` | 保存 AK/SK 到 electron-store（加密） |
| `config:get-status` | renderer→main | 无 | `IpcResult<ConfigStatus>` | 返回已配置/未配置布尔状态 |
| `config:verify-jimeng` | renderer→main | 无 | `IpcResult<void>` | 用存储的密钥执行一次签名验证（不发网络请求） |
| `config:verify-kling` | renderer→main | 无 | `IpcResult<void>` | 用存储的密钥生成 JWT 并验证格式 |

```typescript
// src/shared/types.ts
interface SaveConfigPayload {
  jimengAk?: string
  jimengSk?: string
  klingAk?: string
  klingSk?: string
}

interface ConfigStatus {
  jimengConfigured: boolean
  klingConfigured: boolean
}

interface IpcResult<T = void> {
  success: boolean
  data?: T
  error?: string
}
```

## 3. 即梦 HMAC-SHA256 V4 签名流程

**端点：** `https://visual.volcengineapi.com`
**Region：** `cn-north-1`（固定）
**Service：** `cv`（固定）

签名步骤（4 步派生密钥）：

```
Step 1: 构造 Canonical Request
  POST
  /
  Action=CVSync2AsyncSubmitTask&Version=2022-08-31
  content-type:application/json
  host:visual.volcengineapi.com
  x-content-sha256:{SHA256(body)}
  x-date:{X-Date}

  content-type;host;x-content-sha256;x-date
  {SHA256(body)}

Step 2: 构造 String to Sign
  HMAC-SHA256
  {X-Date}
  {datestamp}/{region}/{service}/request
  {SHA256(canonicalRequest)}

Step 3: 派生签名密钥
  kDate    = HMAC-SHA256(sk, datestamp)
  kRegion  = HMAC-SHA256(kDate, "cn-north-1")
  kService = HMAC-SHA256(kRegion, "cv")
  kSigning = HMAC-SHA256(kService, "request")

Step 4: 计算签名
  signature = HMAC-SHA256(kSigning, stringToSign).toString()

Authorization Header:
  HMAC-SHA256 Credential={ak}/{datestamp}/{region}/{service}/request,
  SignedHeaders=content-type;host;x-content-sha256;x-date,
  Signature={signature}
```

**X-Date 格式：** `20240101T120000Z`（ISO 8601，去掉 `-:` 和毫秒）

## 4. 可灵 JWT HS256 鉴权流程

**端点：** `https://api-beijing.klingai.com`
**Header：** `Authorization: Bearer {token}`

JWT Payload 字段：

| 字段 | 值 | 说明 |
|------|-----|------|
| `iss` | AK | Access Key 作为 issuer |
| `exp` | now + 1800 | 30 分钟有效期 |
| `nbf` | now - 5 | 提前 5 秒生效，防时钟偏差（避免错误码 1003） |

Header：`{ alg: "HS256", typ: "JWT" }`
签名密钥：SK（UTF-8 编码）

**Token 缓存策略：** 缓存 token，`exp - now < 60s` 时重新生成，避免每次请求都签名。

**常见错误码：**
- 1003：Authorization 未到有效时间 → 检查 nbf 是否设置 now-5
- 1004：Authorization 已失效 → 重新生成 token

## 5. electron-store 密钥存储方案

```typescript
// 写入（加密）
const encrypted = safeStorage.encryptString(value)
store.set(key, encrypted.toString('base64'))

// 读取（解密）
const raw = store.get(key) as string
safeStorage.decryptString(Buffer.from(raw, 'base64'))

// 存储键名
// 'jimeng_ak', 'jimeng_sk', 'kling_ak', 'kling_sk'
```

存储位置（Windows）：`%APPDATA%/linglan/config.json`（内容为加密 base64，不可直接读取）
