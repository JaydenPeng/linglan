  <behavior>
    - VideoSubmitParams 包含 prompt、可选 image_url/image_tail_url、duration(3-15)、aspect_ratio、mode
    - VideoMode 枚举: "std" | "pro" | "multi_shot"
    - submitVideoTask 返回 task_id 字符串
    - pollVideoTask 返回 VideoTask，status 为 "submitted"|"processing"|"succeed"|"failed"
    - mode="multi_shot" 时 API 请求体中 mode 字段传 "multi_shot"（不做转换）
    - 图片参数为 base64 时直接传入（可灵 API 支持 base64 data URI）
  </behavior>
  <action>
    1. 创建 src/shared/types/video.ts，导出:
       - VideoMode = "std" | "pro" | "multi_shot"
       - VideoSubmitParams: { prompt, image_url?, image_tail_url?, duration, aspect_ratio, mode: VideoMode }
       - VideoTask: { task_id, status, video_url?, cover_url?, error_message?, created_at }

    2. 创建 src/main/api/klingVideoApi.ts:
       - submitVideoTask(params: VideoSubmitParams, jwtToken: string): Promise<string>
         调用 POST /v1/videos/omni-video，Authorization: Bearer {jwtToken}，返回 task_id
       - pollVideoTask(taskId: string, jwtToken: string): Promise<VideoTask>
         调用 GET /v1/videos/omni-video/{taskId}，返回 VideoTask

    使用 Phase 1 已有的 jwtSigner 生成 token（从 configStore 读取可灵 AK/SK）。
    HTTP 请求使用 Node.js 内置 https 模块（与 Phase 2 图片 API 保持一致）。
