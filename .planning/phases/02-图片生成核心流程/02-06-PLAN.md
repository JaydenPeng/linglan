## Goal

将 Phase 2 已实现但孤立的 UI 组件（CreateView、TaskListView）接入 App.vue 的 Tab 导航，使用户可以访问图片生成功能。

## Context

验证报告（02-VERIFICATION.md）发现核心阻塞：`src/renderer/App.vue` 只渲染 Settings/Video/History 三个 Tab，未引用任何 Phase 2 图片生成组件。`src/views/CreateView.vue` 和 `src/views/TaskListView.vue` 实现完整但处于孤立状态。

项目架构决策（STATE.md）：使用原生 HTML+CSS Tab 切换（v-if），无 vue-router，无 Ionic。

## Tasks

<task id="06-01" title="在 App.vue 中注册图片生成 Tab">

**目标：** 在现有 Tab Bar 中添加「图片」和「任务」两个 Tab，并用 v-if 挂载对应组件。

**步骤：**

1. 打开 `src/renderer/App.vue`
2. 在 `<script setup>` 中导入两个组件：
   ```ts
   import CreateView from '../views/CreateView.vue'
   import TaskListView from '../views/TaskListView.vue'
   ```
3. 将 `activeTab` 类型扩展为包含 `'create'` 和 `'tasks'`：
   ```ts
   const activeTab = ref<'video' | 'create' | 'tasks' | 'history' | 'settings'>('video')
   ```
4. 在 `tabs` 数组中插入两个新 Tab（放在 video 之后，history 之前）：
   ```ts
   { id: 'create' as const, label: '图片', icon: '🖼️' },
   { id: 'tasks' as const, label: '任务', icon: '📋' },
   ```
5. 在 `<main class="page-content">` 中添加对应的 v-else-if 分支：
   ```html
   <CreateView v-else-if="activeTab === 'create'" />
   <TaskListView v-else-if="activeTab === 'tasks'" />
   ```
   放在 `<VideoGeneratePage>` 之后、`<HistoryPage>` 之前。

**验证：**
- App.vue 编译无报错
- 点击「图片」Tab 渲染 CreateView
- 点击「任务」Tab 渲染 TaskListView

</task>

<task id="06-02" title="验证 CreateView 提交后跳转到任务列表">

**目标：** 确认 CreateView 提交任务后能切换到「任务」Tab。

**步骤：**

1. 打开 `src/views/CreateView.vue`，查找提交成功后的跳转逻辑
2. 如果使用 `router.push('/tasks')` 或类似路由跳转 → 替换为事件 emit 或直接操作共享状态
3. 推荐方案：CreateView emit `'navigate'` 事件，App.vue 监听后切换 activeTab：
   ```html
   <!-- App.vue -->
   <CreateView v-else-if="activeTab === 'create'" @navigate="activeTab = $event" />
   ```
   ```ts
   // CreateView.vue 提交成功后
   emit('navigate', 'tasks')
   ```
4. 如果 CreateView 已无路由跳转逻辑（直接依赖 store），则跳过此步骤

**验证：**
- 提交图片任务后自动切换到「任务」Tab（或无跳转逻辑时手动切换可见任务）

</task>
