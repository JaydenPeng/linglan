<task type="auto">
  <name>Task 1: Capacitor 初始化 + 移动端构建配置</name>
  <files>
    capacitor.config.ts,
    vite.mobile.config.ts,
    package.json
  </files>
  <action>
    1. 安装 Capacitor 依赖（如 package.json 中尚未存在）：
       ```bash
       npm install @capacitor/core @capacitor/cli @capacitor/android @capacitor/ios
       npm install @capacitor-community/http @capacitor/preferences @capacitor/filesystem
       ```

    2. 创建 capacitor.config.ts（项目根目录）：
       ```typescript
       import type { CapacitorConfig } from '@capacitor/cli'
       const config: CapacitorConfig = {
         appId: 'com.linglan.app',
         appName: '灵览',
         webDir: 'dist/renderer',
         server: {
           androidScheme: 'https'
         }
       }
       export default config
       ```

    3. 创建 vite.mobile.config.ts（项目根目录）：
       ```typescript
       import { defineConfig } from 'vite'
       import vue from '@vitejs/plugin-vue'
       import { resolve } from 'path'
       export default defineConfig({
         plugins: [vue()],
         resolve: {
           alias: { '@': resolve(__dirname, 'src/renderer') }
         },
         build: {
           outDir: 'dist/renderer',
           emptyOutDir: true,
         },
         base: './',
       })
       ```
       注意：入口使用 src/renderer/index.html（如存在）或 index.html，需确认渲染进程入口路径。

    4. 在 package.json 的 scripts 中添加：
       ```json
       "build:mobile": "vite build --config vite.mobile.config.ts",
       "cap:sync": "npm run build:mobile && npx cap sync"
       ```

    5. 运行 `npm run build:mobile` 验证构建成功，dist/renderer/ 目录生成。

    6. 初始化 Capacitor（如 capacitor.config.ts 已创建则跳过 init）：
       ```bash
       npx cap add android
       npx cap add ios
       ```

    7. 运行 `npx cap sync` 同步 web 资源到原生项目。
  </action>
  <verify>
    <automated>npm run build:mobile</automated>
  </verify>
  <done>dist/renderer/ 目录存在且包含 index.html，android/ 和 ios/ 目录由 cap add 生成，npx cap sync 无报错</done>
</task>

  <files>
    android/app/src/main/AndroidManifest.xml,
    ios/App/App/Info.plist
  </files>
  <action>
    1. 编辑 android/app/src/main/AndroidManifest.xml，在 &lt;manifest&gt; 标签内添加权限（在已有内容基础上追加，不替换）：
       ```xml
       &lt;uses-permission android:name="android.permission.INTERNET" /&gt;
       &lt;!-- Android 12 及以下需要，13+ 不需要 --&gt;
       &lt;uses-permission android:name="android.permission.READ_EXTERNAL_STORAGE"
           android:maxSdkVersion="32" /&gt;
       &lt;uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"
           android:maxSdkVersion="29" /&gt;
       ```

    2. 编辑 ios/App/App/Info.plist，在 &lt;dict&gt; 内添加相册权限描述（在已有内容基础上追加）：
       ```xml
       &lt;key&gt;NSPhotoLibraryAddUsageDescription&lt;/key&gt;
       &lt;string&gt;需要访问相册以保存生成的图片和视频&lt;/string&gt;
       &lt;key&gt;NSPhotoLibraryUsageDescription&lt;/key&gt;

       ```

    3. 再次运行 `npx cap sync` 确保权限配置同步到原生项目。
